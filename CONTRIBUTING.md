# Contributing to WeHunt.gg Feedback

Thank you for helping improve WeHunt.gg. This repository is for product feedback rather than source-code contributions.

## Report a bug

Use the structured [bug report form](https://github.com/ruanbotha33/wehunt-feedback/issues/new?template=bug_report.yml).

A useful bug report includes:

- The affected platform
- The game or achievement involved, when relevant
- Exact steps to reproduce the problem
- Expected and actual behavior
- Browser, device, or operating-system details
- Screenshots, recordings, or non-sensitive logs

Search open and closed issues first. If a matching issue exists, add a reaction or useful reproduction details instead of opening a duplicate.

## Request a feature

Feature requests belong in the [Ideas discussion category](https://github.com/ruanbotha33/wehunt-feedback/discussions/categories/ideas), not in Issues.

Before posting:

1. Search existing Ideas.
2. Upvote a matching request when one exists.
3. Explain the underlying problem and who benefits.
4. Include examples, mockups, or alternatives when useful.

Votes are one input into prioritization. Product fit, impact, implementation effort, platform restrictions, reliability, and maintenance cost also matter.

## Status workflow

Maintainers generally move feedback through these stages:

- **Needs review** — awaiting initial triage
- **Considering** — being evaluated
- **Accepted** — aligned with the product direction
- **Planned** — assigned to a future release or milestone
- **In progress** — actively being implemented
- **Testing** — being validated before release
- **Shipped** — released
- **Declined** — not currently planned

Accepted feature discussions may be converted into Issues and added to the internal project tracker.

## Keep public reports safe

Never post:

- Passwords or recovery codes
- Authentication tokens, cookies, or API keys
- Private account details
- Personal information belonging to another person
- Exploit instructions or undisclosed security vulnerabilities

Use a private contact channel for security-sensitive reports.

## Community behavior

Be respectful and constructive. Keep each report focused on one bug or one idea. Harassment, spam, repeated duplicates, and unrelated promotional content may be removed.
